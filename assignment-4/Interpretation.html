Interpretation

02242: Assignment 04

Christian Gram Kalhauge

Description

In this exercise we are going to build the first semantic analysis, which also happens to be a dynamic analysis. Essentially, we want to create an interpreter for JVM bytecode. Use jvm2json, systematically chose operations that you will allow in your implementation of the JVM.

You should create operational semantics for all implemented operations. The structure is going over a tuple \(\left<\mu, \psi\right>\) representing the machine, where

  1. \(\mu\) is the memory,
    1. which we can access and set:
      1. \(\mu[i]\) means content at index \(i\) (latex : \mu[idx])
      2. \(\mu[i \mapsto v]\) means content at index \(i\) set to \(v\) (latex: \mu[idx \mapsto v]).
      3. Ei., \(\mu[i \mapsto v][i] = v\)
    2. which contains elements from \(V_\mu\).
  2. \(\psi\) is the method stack and consists of elements of \((\lambda, \sigma, \iota)\). Stacks are read from right to left, e.g., \(\psi = \dotsm\psi_2\psi_1\) with \(\psi_1\) being the top element. We’ll denote the empty stack \(\epsilon\). Here,
    1. \(\lambda\) is an array of local variables, indexed and accessed like the memory, but normally only contains 1-10 elements from \(V_\lambda\);
    2. \(\sigma\) is the operational stack (read as the \(\psi\) stack), which contain typed values from \(V_\sigma\).
    3. \(\iota\) is the absolute program counter, containing both the absolute method name \(m\) and the index of the operation \(i\): \(m/i\). Given that \(\iota = m/i\), we’ll just write \(\iota' = \iota/23\) to mean \(\iota' = m/23\), and \(\iota' = \iota + 1\) to mean \(\iota' = m/(i+1)\).

The values are defined1 as: \[ \begin{aligned} V_\mu & := (\mathtt{int}~n) ~|~(\mathtt{float}~f) ~|~(\mathtt{array}~n~a) ~|~(\mathtt{class}~cn~fs) \\ V_\lambda & := (\mathtt{int}~n) ~|~(\mathtt{float}~f) ~|~(\mathtt{ref}~r) \\ V_\sigma & := (\mathtt{byte}~b) ~|~(\mathtt{char}~c) ~|~(\mathtt{short}~s) ~|~V_\lambda \end{aligned} \]

  • \((\mathtt{array}~n~a)\) is an array of \(n\) elements from \(V_\sigma\). \(a\) can be accessed and set like memory. E.g., given \((\mathtt{array}~n~a)\) we can create an array with the 2nd element updated: \((\mathtt{array}~n~(a[2 \mapsto (\mathtt{int}~10)]))\)

  • \((\mathtt{class}~cn~fs)\) is a class with name \(cn\), which has a mapping from absolute field names (field names with class names) to \(V_\sigma\). E.g., creating a class with the field Point.x set to 2.3, is done like this: \((\mathtt{class}~cn~(fs[\texttt{Point.x} \mapsto (\mathtt{float}~2.3)]))\).

The single step semantics should be defined as steps over the state of the machine, in the context of a program \(h, bc\): \[ h, bc \vdash \left<\mu, \lambda, \sigma\right> \to \left<\mu, \lambda, \sigma\right>.\]

In the program we can access the bytecode given the program counter \(bc[\iota]\), and information about the class hieracy using \(h[cn]\).

To get you started, here are three examples. The first \((\texttt{pop}_1)\), pops an element \(s\) of the stack (check the spec here), the second \((\texttt{pop}_2)\) pops two elements. \[ \frac{ b = bc[\iota] \quad b.opr = \texttt{"pop"} \quad b.words = 1 }{ h, bc \vdash \left<\mu, \psi(\lambda, \sigma s_1, \iota)\right> \to \left<\mu, \psi(\lambda, \sigma, \iota + 1) \right> }(\texttt{pop}_1) \] \[ \frac{ b = bc[\iota] \quad b.opr = \texttt{"pop"} \quad b.words = 2 }{ h, bc \vdash \left<\mu, \psi(\lambda, \sigma s_2 s_1, \iota)\right> \to \left<\mu, \psi(\lambda, \sigma, \iota + 1) \right> }(\texttt{pop}_2) \] The rule \((\texttt{invoke}_v)\), does a virtual invocation of the reference. \[ \frac{ \begin{gather} b = bc[\iota] \quad b.opr = \texttt{"invoke"} \quad b.access = \texttt{"virtual"} \\ n = |b.method.args| \quad \mu[r] = (\mathtt{class}~cn~fls) \\ m = \texttt{lookup}_{v}(h, b.method, cn) \\ \lambda'[0] = (\mathtt{ref}~r) \quad \lambda'[1] = \sigma_1 \quad \dotsc \quad \lambda'[n] = (\mathtt{ref}~r) \quad \lambda'[n] = \sigma_n \end{gather} }{ h, bc \vdash \left<\mu, \psi(\lambda, \sigma (\mathtt{ref}~r) \sigma'_1 \sigma'_2 \dotsm \sigma'_n, \iota)\right> \to \left<\mu, \psi(\lambda, \sigma, \iota + 1) (\lambda', \epsilon, m/0) \right> }(\texttt{invoke}_v) \]

Where \(\texttt{lookup}_v\) is a function that finds the absolute method to call given the class hierachy, the virtual method name and class name. Get the full specification here.

Hints

  • Don’t expect to implement the full language, try focus on the operations used in the examples.

  • Start by assuming that \(\psi\) is not a stack but only a single tuple – you can implement method calls later.

  • Create a nice representation of the machine state so you can print it out; by doing this you print the execution trace for all your test cases. This will help you dramatically in developing fast. You can even save a trace to file for each @Case and commit them to source control, that way you ensure you don’t regress (see golden tests).

  • Your implementation should be super dumb, most of your work should be in defining the semantics and then have one-to-one correspondence between:

    class Interpreter:

    def __init__(self, p : Program, verbose : Optional[TextIO]):
        self.program = program
        self.verbose = verbose
        self.memory = {}
        self.stack = []

    def run(self, f : Tuple[Locals, OperStack, ProgramCounter]):
        self.stack.append(f)
        self.log_start()
        self.log_state()
        while self.step():
            self.log_state()
            continue
        self.log_done()

    def step(self):
        (l, s, pc) = self.stack[-1]
        b = self.program.bytecode[pc]
        if hasattr(self, b["opr"]):
            return getattr(self, b["opr"])(b)
        else:
            return false

    def pop(self, b):
        (l, s, pc) = self.stack.pop(-1)
        # Rule (pop_1)
        if b["words"] == 1:
            if len(s) < 1: return False
            self.stack.append((l, s[:-1], pc + 1))
        # Rule (pop_2)
        elsif b["words"] == 2:
            if len(s) < 2: return False
            self.stack.append((l, s[:-2], pc + 1))
        else:
            return False

  • Remember that \(=\) is talking about equality and is not about “setting” stuff. Create new variables if stuff is different. E.g. \(x = x + 1\) is clearly not true, instead write \(x' = x + 1\), and use \(x'\) instead of \(x\).
